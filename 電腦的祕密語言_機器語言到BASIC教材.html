PART 1　電腦怎麼想事情

電腦是一個「開關」的世界

電腦裡面藏著非常非常多的小小開關（叫做電晶體），每一個開關只有兩種狀態，就像教室的電燈開關一樣！

🔛

開

＝ 1

💡

關

＝ 0

小提醒

電腦裡有數十億顆這樣的小開關，透過「開」和「關」不斷組合，就能表達出各種數字、文字，甚至圖片和聲音！

可以請一位小朋友實際去開關教室電燈，讓大家觀察『開』和『關』兩種狀態。強調電腦裡有數十億個這樣的小開關，這是所有電腦運作的最基礎原理。
PART 1　電腦怎麼想事情

第一站：機器語言

🔩

把很多個「開」和「關」排在一起，就變成電腦唯一真正聽得懂的語言——我們叫它「機器語言」，全部都是由 0 和 1 組成的密碼。

01001000  01101001

（這只是示意，代表打招呼的密碼）

✅
對電腦來說：超簡單！
❓
對人類來說：像天書一樣難！

如果要用機器語言寫程式，程式設計師要記住成千上萬串 0 和 1，非常辛苦！

可以把 0 和 1 比喻成摩斯密碼或秘密暗號，增加趣味。強調重點：機器語言是電腦唯一「天生」聽得懂的語言，但對人類非常不友善。
PART 1　電腦怎麼想事情

第二站：組合語言

🏷️

因為機器語言太難記，工程師發明了「組合語言」，用簡短的英文縮寫代替一長串的 0 和 1，方便人類閱讀。

機器語言
01001000
➜
組合語言
MOV A, B

比機器語言好記一點，但要學會很多專業縮寫，還是不容易上手。

強調『組合語言』是機器語言的簡化寫法，一個縮寫（如MOV、ADD）通常對應一小段0與1。可以問小朋友：MOV看起來像哪個英文單字？（move，移動）
PART 1　電腦怎麼想事情

第三站：高階語言

💬

「高階語言」讓程式更接近人類說話的方式，就像用簡單的英文句子跟電腦聊天一樣！

PRINT "Hello!"
IF age > 10 THEN
  PRINT "你長大了！"

這就是我們今天要學的 BASIC！Python、Scratch 也都是高階語言。

強調高階語言最貼近人類語言、最容易學。可以問小朋友：這段程式碼你猜得出來在做什麼嗎？引導他們發現連沒學過程式的人也能猜出大意。

程式語言的階層金字塔

越往上，越像人類講話；越往下，越貼近電腦的 0 與 1 開關世界。

高階語言

組合語言

MOV / ADD　簡短英文縮寫

機器語言

0 與 1　電腦的原生語言

⬆️ 人類容易懂
⬇️ 電腦直接懂
這一頁是總複習圖，把前三頁的內容整合成金字塔。可以帶著小朋友從上往下、或從下往上重新講一次演進故事，確認他們理解順序：機器語言→組合語言→高階語言。

➜

介紹編譯器(compiler)和解譯器(interpreter)的概念，可以簡化說：翻譯官會把我們寫的PRINT、IF這些句子，翻譯成電腦看得懂的0與1，電腦才能真正執行。不需深入區分compiler和interpreter的技術差異，維持在「翻譯」的比喻即可。
PART 2　BASIC 上場

認識 BASIC 語言

🎓

BASIC

全名：Beginner's All-purpose Symbolic Instruction Code
意思是「初學者的萬用符號指令語言」

  • ✓
    專門為初學者設計
    從一開始就是為了讓「完全不會寫程式」的人也能輕鬆上手。
  • ✓
    語法簡單好讀
    很像簡單的英文句子，例如 PRINT、INPUT、IF...THEN。
  • ✓
    程式老手的啟蒙老師
    全世界許多資深工程師，都是從 BASIC 開始學程式的！

"用一句話說完的程式語言"

PRINT "我在學程式！"
BASIC是1964年由達特茅斯學院教授設計的教學語言，重點放在『簡單、好上手』。可以強調：今天大家要學的第一支正式程式語言，就是這個歷史悠久又親切的BASIC。
PART 2　BASIC 上場

BASIC 的指令積木

🖨️
PRINT

叫電腦「說」出文字或數字，顯示在畫面上。

⌨️
INPUT

讓電腦「問問題」，並等待使用者輸入答案。

🧩
LET

幫電腦「記住」一個數字或名字（變數）。

把PRINT、INPUT、LET比喻成三種不同的「積木」，接下來的範例會示範怎麼把這些積木組合起來，變成一支完整的小程式。
PART 2　範例程式

範例：跟電腦打招呼

PRINT "哈囉，大家好！"
執行結果
哈囉，大家好！
怎麼讀懂這句程式？
  • PRINT → 告訴電腦：「請把後面的內容顯示出來」
  • " "（雙引號） → 告訴電腦：「引號裡面是文字，要照樣印出來」
帶大家逐字讀出這一行程式，特別強調雙引號的功用：引號裡面的文字會「原封不動」被印出來。可以讓小朋友試著改成印出自己的名字。
PART 2　範例程式

範例：電腦記住我的名字

INPUT "你叫什麼名字？", name$
PRINT "你好，"; name$; "！很高興認識你"
執行結果（假設輸入「小美」）
你叫什麼名字？ 小美 你好，小美！很高興認識你

1️⃣ 電腦先問問題

2️⃣ 等待你輸入答案

3️⃣ 用 INPUT 記住答案

4️⃣ 用 PRINT 說出結果

說明INPUT會讓程式暫停，等待使用者打字輸入，並把輸入的內容存到name$這個「小盒子」（變數）裡，之後PRINT就能把這個盒子的內容講出來。$符號可以簡單說明是代表『文字』的記號，不需深入細節。
PART 3　動手做

動手做 1：我的第一支程式

🚀

請寫一行程式，讓電腦印出你的自我介紹！

提示模板
PRINT "我是___，今年___歲"
PRINT "我是小安，今年8歲"

範例（可以參考，但要換成你自己的答案喔！）

💻

打開你的 BASIC 編輯器，把程式打進去，按下「Run／執行」按鈕，看看電腦有沒有照你的話說出來！

請每位小朋友實際在電腦上打開BASIC編輯器（見工具介紹頁），練習打出這一行程式並執行。巡場協助拼字或引號漏打的小錯誤，這是初學者最常見的問題。
PART 3　動手做

動手做 2：猜猜我幾歲

🎮

寫一支程式：先問使用者的年齡，再回應一句話！

INPUT "你幾歲？", age
PRINT "哇！"; age; "歲！你已經很棒了！"
挑戰加分（三年級適合）
試著加一句 IF 判斷：
IF age > 10 THEN
  PRINT "你是大孩子了！"
✅

完成後，跟旁邊的同學互相測試看看：換不同的年齡，電腦說的話會不一樣嗎？

「挑戰加分」的IF判斷是給進度較快或三年級孩子的延伸練習，一、二年級可以只做基本的INPUT+PRINT就好，不用勉強加上IF。鼓勵同儕互相測試，增加互動與成就感。

總複習小測驗

❓

1. 電腦真正、天生聽得懂的語言是什麼？

答案：機器語言（0和1組成）

❓

2. PRINT 這個指令的功能是什麼？

答案：叫電腦把文字或數字顯示出來

3. BASIC 語言比較像下面哪一種？

答案：高階語言（最接近人類說話方式）

可以先把答案遮住，用口頭方式讓小朋友搶答，增加互動氣氛，最後再公布答案卡上的解答。也可以讓小朋友舉手表決選項，確認全班理解程度。
